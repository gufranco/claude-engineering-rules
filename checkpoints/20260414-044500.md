## Decisions

- All effects follow readability standard: WHAT/HOW/WHY header comments, descriptive variable names, no single-letter vars except x/y
- 30 fps locked (double swiWaitForVBlank) for all software effects
- ITCM + ARM mode for CPU-heavy inner loops
- 32-bit packed writes for all framebuffer effects
- DMA fill for background clears (starfield, scrolltext, etc.)
- Each effect is self-contained: init/update/destroy, no shared state between effects
- Effects array in demo.c sequences all effects with 480-frame duration (16s at 30fps)
- GPU variants will use `uses_3d` flag in Effect struct, switch MODE_FB0 <-> MODE_5_3D during fade transitions
- Every effect that CAN benefit from GPU MUST have a GPU variant
